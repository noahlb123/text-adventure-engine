#Quick Start
`<h1>Text Adventure Engine</h1>

Welcome to the documentation for the text adventure engine: a simple system to create text based adventures! This documentation explains how to use the system with examples.

<h1>Quick Start</h1>

1. Create a .txt file
2. In the file define a scene name preceded by \`\#\`\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#First Scene</span> 

3. On a newline, define the scene body text between two <span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`</span>\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`Two roads diverged in a yellow wood,</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">And sorry I could not travel both</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">And be one traveler, long I stood</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">And looked down one as far as I could</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">To where it bent in the undergrowth</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">Robert Frost\`</span>

4. Then on newlines below the body, define options. For each option, begin with a \`\-\`, followed by the text the user will see, then a \`\:\` followed by the name of the scene that will result from choosing the option\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#First Scene</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-Take the road less traveled by\: Poem Ending</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-Take the bend in the undergrowth\: Custom Ending</span>

5. After step 4, a scene is complete. The final step is to define new scenes for each option, ensuring that the new scene names exactly match the ones defined in the options. All together this looks like this\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#First Scene</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`Two roads diverged in a yellow wood,</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">And sorry I could not travel both</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">And be one traveler, long I stood</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">And looked down one as far as I could</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">To where it bent in the undergrowth</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">Robert Frost\`</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-Take the road less traveled by\: Poem Ending</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-Take the bend in the undergrowth\: Custom Ending</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;"></span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#Poem Ending</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`And that has made all the difference.</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">Robert Frost\`</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;"></span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#Custom Ending</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`I doubted if I should ever come back.</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">Robert Frost\`</span>

6. At this point you can upload the .txt file to the site and experience the user perspective.

That should be all you need to get started! The rest of the documentation provides comprehensive definitions and examples, but understanding every detail isn't needed. All the code for this project is publicly hosted on <a href=https\://github.com/noahlb123/text\-adventure\-engine>github</a>.`

#Scenes
`<h1>Scenes</h1>
All information in the text adventure engine is organized into scenes. A scene is defined by 3 scene components\: its scene name (which will never be seen by a user), body text, and (not required\:) options.

Requirements\:
* Begin with a scene name
* Scene names must be followed by body text
* If options are included, they must be placed after the body text
* Individual scenes must be separated by newlines
* See special characters section

Example\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#Birthday scene</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`Unfortunately, the clock is ticking</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">the hours are going by. The past increases, the future recedes. Possibilities decreasing, regrets mounting\:</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">Do you understand?\`</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-I understand.\:choose existentialism</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-remain ignorant\:choose ignorance</span>`

#Scene Names
`<h1>Scene Names</h1>
Scene names are the identifiers that connect options to scenes.

Requirements\:
* Must begin with \`\#\`
* Must end with a newline
* Must exactly match the scene name in an option, unless its the first scene defined

Examples\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#Scene 1</span>`

#Body Text
`<h1>Body Text</h1>
Body text is the only thing the user will see in a scene, unless there are options.

Requirements\:
* Must be preceded by a scene name
* Must end and begin with a <span>

Example\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\# Letter from Birmingham Jail</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`I have almost reached the regrettable conclusion that the Negro's great stumbling block in the stride toward freedom is not the White Citizen's Council\-er or the Ku Klux Klanner, but the white moderate who is more devoted to "order" than to justice; who prefers a negative peace which is the absence of tension to a positive peace which is the presence of justice; who constantly says "I agree with you in the goal you seek, but I can't agree with your methods of direct action;" who paternalistically feels he can set the timetable for another man's freedom; who lives by the myth of time and who constantly advises the Negro to wait until a "more convenient season." \-MLK April 1963\`</span>`

#Options
`<h1>Options</h1>
Options are the interactive component of the text engine that connect scenes together.

Requirements\:
* Must be preceded by body text
* Must begin with a \`\-\`
* \`\-\` must be followed by the option text a user will see
* Option text must end with a \`\:\`, and then be followed by a scene name

Example\:
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\#Bridge Keeper's Riddle</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`What is the air\-speed velocity of an unladen swallow?\`</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-40 miles per hour\: Death</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-22 meters per second\: Death</span>
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\-What do you mean? An African or European swallow?\: Bridge Keeper Dies</span>`

#Whitespace
`<h1>Whitespace</h1>
Whitespace does matter in the text engine, but it can also be flexible depending on the circumstance. Specifically, the JavaScript <span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">trim()</span> function is used on each component of a scene, which makes leading and trailing whitespace irrelevant, but means that internal whitespace does matter.`

#Special Characters
`<h1>Special Characters</h1>
The characters used to define each scene component (\`\#\`, <span>, \`\-\`, \`\:\`) cannot be used as normal. Instead these characters must be preceded by the escape character \`\\` in order for them to function properly within the text of a scene component. Additionally, to display \`\\` in text, it must be preceded with an additional \`\\`\:

Proper escape format
\\\#
\\\`
\\\-
\\\:
\\\\

Example body text (will be displayed as \\\`\:\-)
<span style="background\-color\:\#FFFFFF75; font\-family\:Courier New; color\:black;">\`\\\\\\\`\\\:\\\-`</span>`

`<h1>Inline HTML</h1>

For exactly how these elements are added into the user's window, see this project's <a href=https\://github.com/noahlb123/text\-adventure\-engine/blob/main/index.html>index.html code</a>.`
